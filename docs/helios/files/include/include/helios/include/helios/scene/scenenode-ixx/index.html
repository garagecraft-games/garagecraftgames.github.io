33 /**
34 * @brief Represents a SceneNode within a SceneGraph.
35 *
36 * A SceneNode holds its local transformation and a (shared) pointer to
37 * a Renderable object (the representative of a visible element).
38 * Furthermore, a SceneNode can have multiple child nodes uniquely owned by
39 * **this** SceneNode.
40 *
41 * A SceneNode holds a global unique identifier and tracks a dirty state (`needsUpdate_`)
42 * related to any transformations that occurred on the SceneNode.
43 * Ownership is managed by a collection of unique pointers of the child nodes.
44 * SceneNodes are designed to be not copyable and not assignable to prevent duplication
45 * and to enforce the hierarchical tree structure.
46 *
47 * The `worldTransform_` is derived from a SceneNode's `localTransform_` and the SceneNode's
48 * parent's `worldTransform`s. A call to `worldTransform()` will update the particular subtree
49 * of the SceneNode bottom-up to make sure the actual values are available with the SceneNode's
50 * `worldTransform`. A Scene might also propagate changes top-bottom - see `Scene:.udpateNodes()`.
51 *
52 */
53 class SceneNode {

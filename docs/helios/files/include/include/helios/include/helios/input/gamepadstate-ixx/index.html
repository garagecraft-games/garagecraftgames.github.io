19 /**
20 * @brief A lightweight class for transferring the state of a Gamepad.
21 *
22 * This class models a reusable object representing the input state of a gamepad.
23 * The input state of a gamepad consists of the state of the left and right trigger,
24 * ranging from [0, 1], where 1 means fully pressed and 0 means not pressed at all.
25 *
26 * Similarly, a GamepadState provides information about the left and the right
27 * gamepad axes. Both the x- and the y-axis range from [-1, 1]. For the x-axis,
28 * the following holds:
29 * - 0 => not moved at all
30 * - -1 => moved all the way to the left
31 * - 1 => moved all the way to the right
32 *
33 * For the y-axis the positive direction corresponds to "up" and the negative
34 * direction corresponds to "down".
35 *
36 * Updating the values is done by calling `updateAxes` on this object.
37 *
38 * For convenient access of the axes in 2D coordinates, the class provides accessors
39 * to the sticks' axes in `vec2f` form.
40 *
41 * Button states are generally represented with boolean values, i.e. true for pressed,
42 * otherwise false.
43 *
44 * @note Implementations must account for joystick drift: achieving exactly 0
45 * for x/y when no human input occurred is rare. Consider applying a dead zone
46 * when processing the GamepadState. Axis values are individually clipped to [-1.0, 1.0],
47 * but the magnitude of the resulting (x, y) vector may exceed 1.0. Applications should
48 * normalize input vectors where appropriate and possibly offer calibration.
49 *
50 * This implementation is inspired by the GLFW gamepad input model.
51 * @see https://www.glfw.org/docs/latest/input_guide.html#joystick
52 */

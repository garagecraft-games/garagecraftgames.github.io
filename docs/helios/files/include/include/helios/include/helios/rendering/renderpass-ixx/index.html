17 * Represents a single RenderPass under a set of specific uniform values.
18 *
19 * A `RenderPass` holds a `RenderQueue` that contains all `RenderCommands` that need
20 * to be processed in the subsequent rendering.
21 * The associated uniform values specified with this `RenderPass`remain constant
22 * during processing the `RenderCommmand`s.
23 *
24 * `RenderPass`es are typically processed sequentially by an API-specific implementation
25 * of the pure virtual `RenderingDevice`-class.

27 * @note A `RenderPass` should be configurable with other properties relevant to the underlying
28 * rendering backend, e.g. depth testing, draw mode etc.
